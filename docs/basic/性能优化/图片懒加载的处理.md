因为图片资源经常是网页上最大的资源，所以也是有各种文章来介绍如何优化图片加载的性能，觉得一般也就下面的方法了:

- 缩小图片的尺寸
- 图片懒加载

## 缩小图片尺寸

缩小图片，一般有几种方式：

- 使用jpg格式的图片，因为可以设置不同的参数来压缩
- 使用不同的图片格式，webp等
- 使用svg等

一般使用jpg格式的图片居多，因为可以方便的设置图片宽高和透明度，质量等参数，然后可以按照不同的平台自适应。

## 图片懒加载

图片懒加载的基本思路分为如下几步：

- 先展示小图来占位
- 进入视口之后自动加载大图，或者用户点击后加载大图
- 展示大图

有些简单的页面直接不展示占位图片，直接在进入视口后加载图片，但是比较有追求的页面，就在小图片的占位方式上不断进步，提供各种占位方式，只为了更好的用户体验。

参考网页：https://jmperezperez.com/more-progressive-image-loading/

### 小图过度到大图

一般小图过度到大图的时候，如果是比较有追求的网页，小图会使用模糊效果，，，这种模糊效果可以使用css的`filter`(要考虑到兼容性问题)，或者canvas来实现。

### 各种占位: placeholders

占位可以防止图片加载的时候页面高度变化，对用户体验来说是很好的。 这边列一场常见的，其实各种各样的实现方式都有，按需或者按开发者的idea来创造。

- 用空白来占位
- 一些矢量图来占位，比如简单的用户头像等场景
- 固定颜色占位：作为背景色，颜色值最好是基于想要加载的图片，是的图片加载的过度比较平滑
- 模糊图：使用很小尺寸的原图去展位，然后加上模糊的效果，去过渡。
- 基于svg的占位： https://jmperezperez.com/svg-placeholders/
